# Codex-Auth Management Skill

This skill provides all the context an agent needs to manage, run, and troubleshoot the `codex-auth` local proxy. 

## Project Architecture
`codex-auth` is a professional Python package that intercepts ChatGPT via Playwright Stealth to provide an OpenAI-compatible API on `localhost`.
- **Framework:** FastAPI, Uvicorn, Playwright.
- **CLI:** Typer and Rich.
- **Auth Storage:** Local `.codex/auth.json` inside the repository root.

## Installation
If the project is not installed, install it in editable mode:
```bash
pip install -e .
playwright install chromium
```

## Running the Server
The project is completely controlled via the `codex-auth` CLI.
To start the proxy server:
```bash
codex-auth start --port 8000
```
This runs the Uvicorn server and launches the Playwright stealth browser in the background.

## Endpoints for Users
Once the server is running on `http://127.0.0.1:8000`, the following endpoints are available:

1. **Chat Completions:** `POST http://127.0.0.1:8000/v1/chat/completions` (OpenAI format)
2. **Models List:** `GET http://127.0.0.1:8000/v1/models` (OpenAI format)
3. **Ollama Tags:** `GET http://127.0.0.1:8000/api/tags` (Ollama format)

**Client Configuration for standard AI apps (Cursor, ClassSift, etc):**
- **Base URL:** `http://127.0.0.1:8000/v1`
- **API Key:** Any dummy string (e.g., `sk-dummy-key`)
- **Model:** Any model from `/v1/models` (e.g., `gpt-4o`). The server automatically appends `-vision` for vision-capable models to bypass UI restrictions.

## Debugging & Error Handling

If the user reports that the API isn't working or throws an authentication error:

1. **Missing or Expired Tokens:**
   Check if the `.codex/auth.json` file exists and has valid tokens. If it's missing or expired, instruct the user to run the authentication flow:
   ```bash
   codex-auth auth
   ```
   This will pop up a browser for them to log in to ChatGPT.

2. **Server Won't Start:**
   Check the `Rich` logs generated by `codex-auth start`. 
   - If you see `Error loading token`, run `codex-auth auth`.
   - If Playwright throws timeout errors, ensure the user isn't running on a strictly headless environment without proper dependencies, or run `playwright install chromium`.

3. **Missing Features (Vision):**
   The API dynamically pulls model capabilities from the real `https://chatgpt.com/backend-api/models`. If vision isn't working, verify that `product_features.attachments.image_mime_types` is present in the upstream ChatGPT JSON.

**CRITICAL RULE:** Do NOT modify global `.codex` folders outside of this repository. The `auth.json` specifically used by this package is located at `[repository_root]/.codex/auth.json`.
